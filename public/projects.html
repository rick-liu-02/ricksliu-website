AnimeList Duel

HTML, CSS, JS, React, Node.js, MySQL

The premise is simple: Who has wasted more time watching Japanese cartoons?

The solution? A website that pits your MyAnimeList stats head-to-head with another user.

The frontend uses HTML, CSS and React, and the backend uses Node.js and MySQL. Usernames are sent to the backend, which scrapes data from the associated MyAnimeList page. This data is inserted into a database so future requests for the same user are sped up. Since that stored data will slowly go out of date, requests for updated data can also be made. The website cannot function without the Node.js backend, but it can function without the MySQL database.

CamWow!

Python, Git, Agile Design

What if you could set up automated tasks for your computer based on hand gestures?

'CamWow!' was a first-year school project made in Python by a group of five. A Raspberry Pi recognizes hand shapes and fingers and wirelessly sends data to a computer. Based on what the computer gets, certain actions are automatically executed. These actions can be opening computer files or opening links with optional login info (which was implemented with Selenium), and the user can customize what each gesture will do with a GUI.

Of course, I only worked on 20% of this. Specifically, I was responsible for connecting the Pi to the computer, implementing automatic file opening, saving actions to persistent files, and creating a GUI to edit said files.

Still confused? Unfortunately, the source code can never see the light of day, but check out the totally unbiased video showcasing the project.
